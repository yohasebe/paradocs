詳しい仕様

1. キー・バインディング

プレゼン中の様々な機能はマウスを用いて実行できますが、キーボードを使った操作も可能です。

↓ 次 のアイテムに移動
↑ 前 のアイテムに移動
j 次 のアイテムに移動
k 前 のアイテムに移動
SPACE 次 のアイテムに移動
SHIFT+SPACE 前 のアイテムに移動
. or b 合成音声の開始/停止; 動画・音声の開始/停止（YouTube 動画を含む）; ポップアップ画像の拡大/縮小

便利な使い方

Logitech Wireless Presenter R400/R800 というプレゼン用ポインターデバイスはおすすめです。Paradocs でのプレゼン中のアイテム間の移動や、音声合成・動画・音声の開始/停止に、物理ボタンを使うことができます。

2. いくつかの基本的概念

センテンス（sentence）

スライド上ではパラグラフ内の隣り合った要素として扱われます。スライド上で次々とハイライトされていきます。センテンスを記述する際には単一の改行↩によって区切ります。ただし、自動センテンス分割機能を使うと、用いている言語に応じて自動でセンテンスの分割が行われます。自動センテンス分割を行う際には、テキストの頭に!!を付けます。

パラグラフ（paragraph）

複数のセンテンスから構成されるブロックのことをパラグラフと呼びます。1行1センテンスで記述していきます。

ブロック（block）

ブロックは2つ以上の連続した改行↩↩によって区切られます。1つのブロックは複数の要素を持つことができますが、複数のセンテンスによって構成されたブロックをパラグラフと呼びます。

スライド（slide）

スライドは「ページ」と呼ぶこともできます。スライドは4つ以上のハイフンと改行（----↩ または - - - -↩）で区切ります。1つのスライドは複数のブロックを持つことができます。

デッキ（deck）

デッキはスライドのグループです。4つ以上のイコール記号（====）で区切ります。複数のデッキを含むプレゼンテーションでは、各デッキ内のスライドはReveal.jsの同一水平セクション内で縦に配置されます。

パラグラフの記述

センテンスを改行しながら縦に記述していくとパラグラフ（ブロック）になります。

----
Sentence #1 
Sentence #2  
Sentence #3 
----

自動センテンス分割機能を使うと、上記は下のように書くことができます。

----
!! Sentence #1 Sentence #2 Sentence #3
----

自動センテンス分割を行うには、テキストの頭に!!を付けます。また、各センテンスの最後には文区切りの記号（英語ならピリオド、日本語なら句点など）が付いている必要があります。

複数ブロックの記述

ブロックはパラグラフ（段落）に対応します。ただし、Paradocsでは「1スライドにつき1ブロック」を推奨します。

----
Block #1 Sentence #1
Block #1 Sentence #2

Block #2 Sentence #1
Block #2 Sentence #2
----

複数スライドの記述

スライドは「ページ」に対応します。1つのスライドに複数のブロックを含められます。

----
Slide #1
----
Slide #2
----
Slide #3
----

3. ブロックの種類

ここでは様々な種類のブロックについて示します。

見出し

異なるレベルの見出しを付けることができます。見出しは選択したアクセント・カラーで表示されます。

----
# Heading 1 

## Heading 2 

### Heading 3 
----

通常のパラグラフ

パラグラフは1行1センテンスで記述します。

This is regular paragraph. 
Sentences in static text are highlighted when focused. 
They are printed in grey when not focused.

通常のパラグラフでは、キーを押すことでセンテンスごとにハイライトが移動します。スライドに収まりきらないときには自動でスクロールされます。

静的なパラグラフ

静的なパラグラフを作成するには、パラグラフを構成する各センテンスの行頭に垂直バー記号とスペースを配置します。

| This is static paragraph. 
| Sentences in static text are not highlighted 
| They are always printed in black.

または

| This is static paragraph. 
  Sentences in static text are not highlighted 
  They are always printed in black.

静的なパラグラフでは、センテンスごとのハイライト移動がオフになります。スライドに収まらないときにも自動でスクロールされないので注意が必要です。

順序なしリスト

いわゆる箇条書きの書式です。アスタリスクの後にスペースが必要です。スライドに収まりきらないときには自動でスクロールされます。

* Unordered list item 1
* Unordered list item 2 
* Unordered list item 3

順序付きリスト

番号付きの箇条書きの書式です。番号の後にスペースが必要です。スライドに収まりきらないときには自動でスクロールされます。

1. Ordered list item 1
2. Ordered list item 2
3. Ordered list item 3

a. Ordered list item 1
b. Ordered list item 2
c. Ordered list item 3

注： 箇条書きの最初の番号を指定することはできません。

番号付きブロック

任意の番号を付けたブロックを作成することができます。問題集の公式や問題を提示するようなときに便利です。

365. The number in a numbered block will be printed only once; 
     the left-hand side of the text can have multiple lines 
     the lines are aligned nicely

注： 上記の例で2行目以降の左側のスペースは重要です。

4. 文字装飾とノート

ここでは文字装飾と、本文の近くに付加的に表示するノート（注記テキスト）の作成方法を説明します。

文字装飾

太字、斜字、下線、ハイライトが使用可能です。

斜字:       orange *strawberry* apple
太字:       orange **strawberry** apple
下線:       orange _strawberry_ apple
ハイライト: orange ==strawberry== apple

Markdown互換性

Paradocsは独自の書式ルールに加えて、Markdown構文の一部をサポートしています。以下の表に、対応状況と標準Markdown（GFM）との違いをまとめます。

機能 Paradocs構文 標準Markdown 備考
太字 **text** **text** 同一構文
斜字 *text* *text* 同一構文
下線 _text_ _text_（斜字） Paradocsでは下線として扱います
ハイライト ==text== 非対応 Paradocs独自拡張
取り消し線 ~~text~~ ~~text~~ GFM互換
テーブル | 列 | 列 | | 列 | 列 | GFM互換；marked.jsでレンダリング
リンク [テキスト](url) [テキスト](url) 同一構文
画像 ![alt](url) ![alt](url) 同一構文；![alt](local:ファイル名) でアップロード画像も使用可
見出し # 見出し # 見出し 同一構文（レベル1〜3）
順序なしリスト * 項目 * 項目 同一構文
順序付きリスト 1. 項目 1. 項目 同一構文；a. 項目 形式にも対応
インラインコード `code` `code` 同一構文
引用 > テキスト > テキスト 同一構文
コードブロック ``` ``` フェンスドコードブロック；言語指定（例: ```python）は無視されます

注： Paradocsはプレゼンテーション用途に特化した独自のテキストパーサーを持っています。一部の構文はMarkdownと共通ですが、以下の点にご注意ください：

  • _text_ は下線になります（斜字ではありません。斜字には *text* を使用）
  • ----（4文字以上のハイフン）は水平線ではなくスライド区切りです
  • ====（4文字以上のイコール）はデッキ区切り（スライドのグループ分け）です
  • 生のHTMLタグ（<b>、<div> など）はエスケープされプレーンテキストとして表示されます
  • ネストされたリストには対応していません（フラットなリストのみ使用可能）
  • Setext形式の見出し（タイトルの下に === や ---）は非対応です。ATX形式（# タイトル）を使用してください
  • フルスライドのメディア表示にはParadocs専用のコマンド（image:、video: など）を使用してください

ノートとポップアップ画像

ノートは波カッコとnoteコマンドを用いて作成します。文字によるノートのほか、画像のURLを指定してポップアップさせることができます。

This is part of main text. {note: これは注です}
This is also part of main text. {note: これも注です}
You can also add a pop-up image. {image: url_to_your_image_file.(png|jpg|gif)}

注： ノートとポップアップ画像は「通常のパラグラフ」と「順序なし／順序付きリスト」で利用可能です。なお、自動センテンス分割機能を用いる場合は、ノートとポップアップ画像を表示できません。波括弧{ }とその中のテキストは削除されます。

便利な使い方

自分で作成した図形などを埋め込みたい場合、Google Drawingsで作成して ファイル ウェブに公開 で取得したPNG画像のリンクを貼り付けることができます。フローチャートやガントチャートなどのダイアグラムを埋め込みたい場合は、Mermaid Live Editorの Link to SVG でSVGのリンクを取得して貼り付けることができます。

クイズ

最初はオーディエンスから隠しておいて、適当なタイミングで答を示すようなクイズを記述することができます。

| In a static context, words and phrases in {curly brackets}
| will be rendered as {quiz items} overlayed by an opaque box.

注： クイズを記述する際には、静的なパラグラフもしくは静的な順序つき／順序なしリストを用いる必要があります。

多肢選択式クイズ（MCQ）

クリック可能な選択肢付きクイズを作成できます。プレゼンターが選択肢をクリックすると正誤が表示されます。MCQブロックは静的パラグラフ（|で始まる行）内に配置します。

| {mcq: フランスの首都はどこですか？
|   a) ロンドン
|   b) ベルリン
|   *c) パリ
|   d) ローマ
| }

正解の選択肢には*を付けます。正解をクリックすると緑でハイライトされます。不正解をクリックするとその選択肢だけが赤でハイライトされ、やり直しのメッセージが表示されます。正解は明かされないので、何度でも挑戦できます。正解後にはやり直しボタンが表示され、クイズをリセットできます。

テーブル

Markdownのテーブル構文を使用してテーブルを作成できます。すべての行が|で始まり|で終わる必要があり、2行目はセパレータ行です。

| 名前   | 点数  |
|--------|-------|
| Alice  | 95    |
| Bob    | 87    |

注： 行頭と行末の両方に|があるテーブル行は、行頭のみに|がある静的パラグラフと区別されます。

5. メディア・ファイルの埋め込み

画像、動画、音声などをスライドに埋め込むことができます。

画像の埋め込み

ウェブ上にアップロードされたPNGやJPGなどのファイルURLを用いて画像を指定します。画像はスライド全体に可能な限りで拡大表示されます。

image: url_to_your_image_file.png

または

img: url_to_your_image_file.png

注： 画像を表示するには、上記コマンドがそのスライド中で唯一のブロックとなるように記述します。複数のブロックを記述した場合には、画像自体は表示されず、画像へのリンクが用いられます。

ローカル画像

エディタ横のドロップゾーンを使って、デバイスから画像をアップロードできます。対応形式：JPEG、PNG、GIF、WebP（各5MBまで）。アップロードされた画像はブラウザのローカルストレージに保存され、ページのリロードやブラウザの再起動後も保持されます。すべてリセットボタンをクリックするか、ブラウザのデータを手動で削除すると消去されます。

ローカル画像をスライド全体に表示するには、画像リストのファイル名をクリックするか、以下のように入力します：

image: local:your_image_file.jpg

Markdownのインライン画像構文も使用できます：

![説明](local:your_image_file.jpg)

YouTube 動画の埋め込み

YouTubeのURLを用いて動画を指定します。

youtube: https://www.youtube.com/watch?v=Ks-_Mh1QhMc

または

yt: https://www.youtube.com/watch?v=Ks-_Mh1QhMc

youtube動画のURLには開始時点と終了時点を指定することができます。その際には &start=x&end=y の形式でオプションをURLに追加します。動画全体の中でスライドで再生するセグメントは動画の始まりからの経過秒数で指定します。下記の指定では動画の30秒地点（x=30）から60秒地点（y=60）までを再生します。

youtube: https://www.youtube.com/watch?v=MMmOLN5zBLY&start=30&end=60

もっと直感的な 時間:分:秒 の形式で指定することもできます。

youtube: https://www.youtube.com/watch?v=MMmOLN5zBLY&start=0:30&end=1:00

注： YouTube 動画を表示するには、上記コマンドがそのスライド中で唯一のブロックとなるように記述します。複数のブロックを記述した場合には、動画自体は表示されず、動画へのリンクが用いられます。

MP4 動画の埋め込み

ウェブ上にアップロードされたMP4形式ファイルのURLを用いて動画を指定します。

video: url_to_your_video_file.mp4

MP4動画のURLには開始時点と終了時点を指定することができます。その際には #t=x,y の形式でオプションをURLに追加します。それぞれ、動画の最初からの経過秒数で表すことに注意してください。下記の指定では動画の最初（x=0）から5秒後（y=5）までを再生します。

video: url_to_your_video_file.mp4#t=0,5

注： MP4動画を表示するには、上記コマンドがそのスライド中で唯一のブロックとなるように記述します。複数のブロックを記述した場合には、動画自体は表示されず、動画へのリンクが用いられます。

MP3 音声の埋め込み

ウェブ上にアップロードされたMP3形式ファイルのURLを用いて音声を指定します。

audio: url_to_your_audio_file_1.mp3

audio: url_to_your_audio_file_2.mp3

MP3音声ファイルのURLには開始時点と終了時点を指定することができます。その際には #t=x,y の形式でオプションをURLに追加します。それぞれ、音声の最初からの経過秒数で表すことに注意してください。下記の指定では音声の最初（x=0）から5秒後（y=5）までを再生します。

audio: url_to_your_audio_file.mp3#t=0,5

注： 1つのスライドの中で音声ファイルのブロックを複数記述することが可能です。スライドに収まりきらないときには自動でスクロールされます。

6. その他の機能

テキストの音声合成

通常のパラグラフ内のセンテンスはブラウザから利用可能な音声合成エンジンで音声を再生することができます。使用可能な言語と音声のバリエーションは OS とブラウザによって異なります。

音声合成が可能なときには、プレゼン画面の左上にスピーカーのアイコンが現れます。このアイコンをクリックするか、もしくはキーボードの . キーを打鍵すると音声が再生されます。もう一度クリック／打鍵すると再生を途中で停止することができます。

音声再生中は、現在読み上げられている単語が黄色の背景でハイライトされます。このワードレベルのハイライトにより、オーディエンスはテキストの読み上げ箇所を追いやすくなります。再生が終了または停止されると、ハイライトは自動的にクリアされます。

注： ワードレベルのハイライトはブラウザのonboundaryイベントのサポートに依存します。ChromeとEdgeでは動作しますが、すべてのブラウザで利用可能とは限りません。

スティッキー・ノート

プレゼン中に思いついた内容をスティッキー・ノートに書いてオーディエンスに示すことができます。

プレゼン画面左上の「紙切れ」アイコンをクリックするとスティッキー・ノートの表示／非表示を切り替えられます。スティッキー・ノートの内容はスライドが切り替わってもそのまま保持されます。

レーザーポインター

プレゼン中のマウスポインターの形状をレーザーポインター風に切り替えることができます。プレゼン画面左上の●アイコンをクリックしてください。

レーザー風にしたときのポインターの色は設定画面で指定したアクセント・カラーと同じになります。

自動プレゼン機能

画面左上の魔法の杖アイコンをクリックしてください。現在のスライドの次の断片から発表全体の最終の断片まで、テキストの読み上げや動画再生などが次々と自動で実行されます。

自動保存

入力テキスト、フォーム設定、アップロードしたローカル画像はブラウザのローカルストレージに自動的に保存されます。ブラウザを閉じた後でも、ページを再訪問すると前回の作業内容が自動復元されます。クリアでエディタのテキストのみ消去、すべてリセットでテキスト・設定・アップロード画像のすべてを消去できます。

ファイルメニュー（保存 / 読込 / ダウンロード）

ファイルドロップダウンボタンからファイル操作ができます：

  • テキストを保存 — エディタの現在のテキストを.txtファイルとしてダウンロードします（paradocs-source.txt）。
  • テキストを読込 — ファイルピッカーから.txtまたは.mdファイルを選択してエディタに読み込みます。ファイルの種類（テキストのみ）、サイズ（最大1MB）、内容の安全性（スクリプトタグ・イベントハンドラの除去）が検証されます。
  • HTMLをダウンロード — プレゼンテーションをスタンドアロンHTMLファイルとしてエクスポートします（下記参照）。

HTMLダウンロード

ファイルメニューのHTMLをダウンロードをクリックすると、プレゼンテーションをスタンドアロンのHTMLファイルとしてエクスポートできます。スライドの内容、設定、スタイルがすべてファイル内に埋め込まれるため、Paradocsのウェブサイトにアクセスしなくてもブラウザで開くことができます。外部ライブラリ（jQuery、Reveal.js等）はCDN経由で読み込まれます。

注意 エクスポートされたHTMLファイルでは、YouTube動画の埋め込みはクリック可能なサムネイルリンクに自動変換されます。これはローカルHTMLファイルではブラウザのセキュリティ制限によりYouTubeのiframe埋め込みが機能しないためです。サムネイルをクリックするとYouTubeで動画が開きます。

スライド位置と内部リンク

プレゼン中にスライドやフラグメントを移動すると、ブラウザのURLが自動的に更新されます（例：#/0/1/3）。このハッシュベースのURLを使って、スライド間の内部リンクを作成できます。例えば <a href="#/2">スライド3へ</a> のようなアンカータグで、プレゼン内の別のスライドにジャンプできます。

フィルムストリップ・プレビュー

アクションボタンの横にあるプレビューボタン（目のアイコン）をクリックすると、エディタの右側にフィルムストリップパネルが表示されます。テキスト入力中にスライドのサムネイルが自動更新されます（800msの遅延あり）。

  • 同期モードON（デフォルト）: サムネイルをクリックすると、エディタのカーソルが該当スライドのテキスト位置にジャンプし選択されます。
  • 同期モードOFF: サムネイルをクリックすると、ライトボックスでスライドが拡大表示されます。矢印キーでスライド間を移動、Escapeまたは外側クリックで閉じます。
  • エディタヘッダーの同期トグル（リンクアイコン）を有効にすると、カーソル位置に合わせてフィルムストリップが自動スクロールします。
  • エディタとパネルの間のリサイズハンドルをドラッグして、フィルムストリップの幅を調整できます。

入力中にパースエラーが発生した場合、プレビューは最後に正常にレンダリングされた内容を維持します。

注： フィルムストリップパネルはブラウザのウインドウ幅が900pxを超える場合にのみ利用可能です。それより小さい画面ではプレビューボタンは自動的に非表示になります。仮想スクロールにより、スライド数が多くてもメモリ使用量を低く抑えます。

オーバービュー・モード

プレゼン中にESCキーを押すと、オーバービュー・モードに切り替わります。すべてのスライドがグリッド状にサムネイル表示され、大量のスライドがあっても素早くナビゲートできます。

  • 矢印キー（←→↑↓）で選択を移動できます。
  • Enterで選択したスライドに移動し、プレゼンに戻ります。
  • 再度ESCを押すとオーバービューを閉じて現在のスライドに戻ります。
  • サムネイルをクリックすると、そのスライドに直接移動します。

YouTube動画のスライドは埋め込みプレーヤーの代わりにサムネイル画像が表示されます。プレゼン画面右上のグリッドアイコンからもオーバービューを切り替えられます。

ビープ音

プレゼン画面左側ツールバーのベルアイコンをクリックすると、フラグメント（文やリスト項目）が進むたびに短いビープ音が鳴ります。オーディエンスにステップの進行を知らせるのに役立ちます。もう一度クリックすると無効になります。

色の反転

文字/背景の色を逆転チェックボックスをオンにすると、ハイライト部分の表示が反転します。通常は透明背景に色付きテキストですが、反転時はハイライト色が背景として塗りつぶされ、テキストが白色になります。
