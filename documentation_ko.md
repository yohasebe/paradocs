### 1. 키 바인딩

프레젠테이션 중 사용 가능한 다양한 기능은 마우스로 실행할 수 있지만, 키보드로도 조작할 수 있습니다.


| 키                        | 기능                                                                          |
|:--------------------------|:------------------------------------------------------------------------------|
| `↓`                      | **다음** 항목으로 이동                                                        |
| `↑`                      | **이전** 항목으로 이동                                                        |
| `j`                       | **다음** 항목으로 이동                                                        |
| `k`                       | **이전** 항목으로 이동                                                        |
| `SPACE`                   | **다음** 항목으로 이동                                                        |
| `SHIFT+SPACE`             | **이전** 항목으로 이동                                                        |
| `.`                       | **TTS 음성 읽기** 재생/정지; **동영상/오디오 클립** 재생/정지(YouTube 동영상 포함); **팝업 이미지** 확대/축소 |
| `a`                       | **자동 프레젠테이션** 시작/정지                                               |
| `/`                       | **블랙아웃**(화면 암전) 시작/해제                                             |
| `f`                       | **전체 화면 모드** 시작                                                       |
| `ESC`                     | **개요 모드** 시작; **전체 화면 모드** 종료                                   |
| `s`                       | **메모지** 표시/숨기기                                                        |
| `p`                       | **레이저 포인터** 모드 활성화/비활성화                                        |
| `SHIFT+TAB`               | **메모지** 편집 종료(포커스 해제)                                             |
| `g`                       | 새 브라우저 탭에서 **Google**을 열고 현재 선택한 텍스트 검색                  |

**활용 팁**

> 프레젠테이션 포인터 장치인 [Logitech Wireless Presenter R400/R800](https://www.logitech.com/en-us/presenters)을 추천합니다. Paradocs에서 프레젠테이션할 때 물리적 버튼으로 항목 간 이동, TTS 음성 읽기, 동영상 및 오디오의 시작/정지를 수행할 수 있습니다.

### 2. 기본 개념

**문장(Sentence)**

문장은 단일 줄바꿈(`↩`)으로 구분됩니다. 단락 내에서 문장은 인접한 요소로 취급되며, 차례로 하이라이트됩니다.
자동 문장 분할 기능을 사용하면 사용 언어에 따라 텍스트를 자동으로 문장 단위로 분할할 수 있습니다. 자동 문장 분할을 수행하려면 텍스트 앞에 `!!`를 붙입니다.

**단락(Paragraph)**

여러 문장으로 구성된 블록을 단락이라고 합니다. 단락은 한 줄에 하나의 문장을 포함하는 여러 줄로 이루어진 블록으로 표현됩니다.

**블록(Block)**

블록은 두 번 이상의 연속 줄바꿈 `↩↩`으로 구분됩니다. 하나의 블록에는 둘 이상의 요소를 포함할 수 있습니다. 여러 문장으로 구성된 블록을 특별히 단락이라고 합니다.

**슬라이드(Slide)**

슬라이드는 "페이지"라고도 부를 수 있습니다. 슬라이드는 네 개 이상의 하이픈 뒤에 줄바꿈(`----↩` 또는 `- - - -↩`)으로 구분됩니다. 하나의 슬라이드에는 둘 이상의 블록을 포함할 수 있습니다.

#### 단락 작성 방법

여러 문장을 줄바꿈으로 세로로 나열하면 단락이 됩니다.

```text
----
Sentence #1
Sentence #2
Sentence #3
----
```

자동 문장 분할 기능을 사용하면 위의 내용을 다음과 같이 작성할 수 있습니다.

```text
----
!! Sentence #1 Sentence #2 Sentence #3
----
```

자동 문장 분할을 수행하려면 텍스트 앞에 `!!`를 붙입니다. 또한 각 문장은 마침표 등의 문장 구분 부호로 끝나야 합니다.


#### 슬라이드에 여러 블록 배치하기

하나의 슬라이드에 둘 이상의 블록을 배치하려면 블록 사이에 빈 줄을 하나 넣습니다.

```text
----
Block #1 Sentence #1
Block #1 Sentence #2

Block #2 Sentence #1
Block #2 Sentence #2
----
```

#### 여러 슬라이드(페이지) 만들기

슬라이드는 페이지에 해당합니다. 하나의 슬라이드에 여러 블록을 포함할 수 있습니다.

```text
----
Slide #1
----
Slide #2
----
Slide #3
----
```

### 3. 블록의 종류

다양한 종류의 블록에 대해 설명합니다.

#### 제목

여러 수준의 제목을 만들 수 있습니다. 제목은 선택한 강조 색상으로 표시됩니다.

```text
----
# Heading 1

## Heading 2

### Heading 3
----
```

#### 일반 단락

일반 단락은 한 줄에 하나의 문장을 작성하여 만듭니다.

```text
This is regular paragraph.
Sentences in static text are highlighted when focused.
They are printed in grey when not focused.
```

일반 단락에서는 키를 누르면 문장별로 하이라이트가 이동합니다. 텍스트가 슬라이드에 다 들어가지 않으면 자동으로 스크롤됩니다.

#### 정적 단락

정적 단락을 만들려면 단락을 구성하는 각 문장의 맨 앞에 세로줄 기호와 공백을 배치합니다.

```text
| This is static paragraph.
| Sentences in static text are not highlighted
| They are always printed in black.
```

또는

```text
| This is static paragraph.
  Sentences in static text are not highlighted
  They are always printed in black.
```

정적 단락에서는 문장별 하이라이트가 비활성화됩니다. 정적 단락은 슬라이드에 다 들어가지 않아도 자동 스크롤되지 않으므로 주의가 필요합니다.

#### 순서 없는 목록

이른바 글머리 기호 형식입니다. 순서 없는 목록의 항목을 기술하려면 각 줄의 맨 앞에 별표와 공백을 넣습니다.

```text
* Unordered list item 1
* Unordered list item 2
* Unordered list item 3
```

목록의 모든 항목이 슬라이드에 들어가지 않으면 자동으로 스크롤됩니다.

#### 순서 있는 목록

순서가 있는 목록 형식입니다. 이 유형의 목록 항목을 기술하려면 각 줄의 맨 앞에 번호/문자, 마침표, 공백을 넣습니다.

```text
1. Ordered list item 1
2. Ordered list item 2
3. Ordered list item 3

a. Ordered list item 1
b. Ordered list item 2
c. Ordered list item 3
```

목록의 모든 항목이 슬라이드에 들어가지 않으면 자동으로 스크롤됩니다.

**참고:** 목록의 시작 번호/문자를 지정할 수는 없습니다.

#### 번호 지정 블록

지정한 번호를 가진 블록을 만들 수 있습니다. 프레젠테이션에서 공식이나 연습 문제를 제시할 때 유용합니다.

```text
365. The number in a numbered block will be printed only once;
     the left-hand side of the text can have multiple lines
     the lines are aligned nicely
```

**참고:** 위 예시에서 두 번째 줄 이후의 왼쪽 공백은 중요합니다.

### 4. 텍스트 장식 및 주석

#### 텍스트 장식

텍스트를 굵게, 기울임꼴, 밑줄, 하이라이트로 꾸밀 수 있습니다.

```text
Italic:    orange *strawberry* apple
Bold:      orange **strawberry** apple
Underline: orange _strawberry_ apple
Highlight: orange ==strawberry== apple
```

#### Markdown 호환성

Paradocs는 자체 서식 규칙과 함께 Markdown 구문의 일부를 지원합니다. 아래 표에 지원 현황과 표준 Markdown(GFM)과의 차이점을 정리했습니다.

| 기능 | Paradocs 구문 | 표준 Markdown | 비고 |
|:-----|:--------------|:--------------|:-----|
| 굵게 | `**text**` | `**text**` | 동일 구문 |
| 기울임꼴 | `*text*` | `*text*` | 동일 구문 |
| 밑줄 | `_text_` | `_text_`(기울임꼴) | Paradocs에서는 밑줄로 사용 |
| 하이라이트 | `==text==` | 해당 없음 | Paradocs 확장 기능 |
| 표 | `\| 열 \| 열 \|` | `\| 열 \| 열 \|` | GFM 호환; [marked.js](https://marked.js.org/)로 렌더링 |
| 제목 | `# 제목` | `# 제목` | 동일 구문(레벨 1~3) |
| 순서 없는 목록 | `* 항목` | `* 항목` | 동일 구문 |
| 순서 있는 목록 | `1. 항목` | `1. 항목` | 동일 구문; `a. 항목` 형식도 지원 |

**참고:** Paradocs는 프레젠테이션 용도에 특화된 자체 텍스트 파서를 갖추고 있습니다. 일부 구문은 Markdown과 공통이지만 다음 표준 Markdown 기능은 **지원되지 않습니다**: 인라인 코드(`` `code` ``), 인용(`>`), 이미지(`![alt](url)`), 중첩 목록. 미디어 삽입에는 Paradocs 전용 명령어(`image:`, `video:` 등)를 사용하세요.

#### 노트 및 팝업 이미지

본문과 분리된 짧은 텍스트를 노트로 표시할 수 있습니다. 노트는 중괄호와 `note` 명령어를 사용하여 만듭니다. 텍스트 노트 외에도 `image` 명령어로 파일 URL을 지정하여 이미지 팝업을 만들 수 있습니다.

```text
This is part of main text. {note: This is a note}
This is also part of main text. {note: This is another note}
You can also add a pop-up image. {image: url_to_your_image_file.(png|jpg|gif)}
```

**참고:** 노트와 팝업 이미지는 "일반 단락"과 "순서 없는/있는 목록"에서 사용할 수 있습니다(정적 단락에서는 사용 불가). 자동 문장 분할 기능을 사용할 때는 노트와 팝업 이미지를 표시할 수 없습니다. 중괄호 `{ }`와 그 안의 텍스트는 무시됩니다.

#### 퀴즈

중괄호와 `quiz` 명령어를 사용하여 처음에는 청중에게 텍스트를 숨기고 적절한 시점에 공개하는 퀴즈를 만들 수 있습니다.

```text
| In a static context, words and phrases in {curly brackets}
| will be rendered as {quiz items} overlayed by an opaque box.
```

**참고:** 퀴즈를 만들 때는 정적 단락 또는 정적 순서 있는/없는 목록을 사용해야 합니다.

#### 객관식 퀴즈(MCQ)

청중이 클릭하여 답할 수 있는 객관식 퀴즈를 만들 수 있습니다. MCQ 블록은 정적 단락(`|`로 시작하는 줄) 안에 배치합니다.

```text
| {mcq: What is the capital of France?
|   a) London
|   b) Berlin
|   *c) Paris
|   d) Rome
| }
```

정답 선택지 앞에 `*`를 붙입니다. 선택지를 클릭하면 정답은 녹색, 오답은 빨간색으로 하이라이트되며 정답이 표시됩니다. 답변 후 **다시 풀기** 버튼이 나타나 퀴즈를 초기화하고 다시 도전할 수 있습니다.

#### 표

표준 Markdown 표 구문을 사용하여 표를 만들 수 있습니다. 모든 줄은 `|`로 시작하고 `|`로 끝나야 하며, 두 번째 줄은 구분선이어야 합니다.

```text
| Name  | Score |
|-------|-------|
| Alice | 95    |
| Bob   | 87    |
```

**참고:** 줄의 시작과 끝 모두에 `|`가 있는 표 행은 줄 시작에만 `|`가 있는 정적 단락과 구분됩니다.

### 5. 미디어 파일 삽입

슬라이드에 이미지, 동영상, 오디오를 삽입할 수 있습니다.

#### 이미지 삽입

웹에 업로드된 PNG/JPG/GIF 형식의 이미지 파일 URL을 지정하여 슬라이드에 표시할 수 있습니다. 이미지는 가능한 한 크게 확대됩니다.

```text
image: url_to_your_image_file.png
```

또는

```text
img: url_to_your_image_file.png
```

**참고:**
이미지를 슬라이드에 표시하려면 위 명령어가 해당 슬라이드에서 유일한 블록이어야 합니다. 여러 블록을 기술하면 이미지 자체는 표시되지 않고 이미지 링크가 사용됩니다.

**활용 팁**

> 직접 만든 도형을 삽입하려면 [Google Drawings](https://docs.google.com/drawings/u/0/create)에서 생성한 후 `웹에 게시` 기능으로 생성된 이미지 링크를 붙여넣을 수 있습니다. 또는 [Mermaid Live Editor](https://mermaid-js.github.io/mermaid-live-editor)의 `Link to SVG`를 사용하여 순서도나 간트 차트 같은 비즈니스 다이어그램을 삽입할 수도 있습니다.

#### YouTube 동영상 삽입

YouTube 동영상의 URL을 지정하여 슬라이드에 표시할 수 있습니다. 동영상은 가능한 한 크게 확대됩니다.

```text
youtube: https://www.youtube.com/watch?v=Ks-_Mh1QhMc
```

또는

```text
yt: https://www.youtube.com/watch?v=Ks-_Mh1QhMc
```

YouTube 동영상의 URL에 시작 지점과 종료 지점을 지정할 수 있습니다. 동영상 시작 시점부터의 경과 초수로 지정하며, `&start=x&end=y` 형식으로 URL에 옵션을 추가합니다. 다음 예시는 동영상의 30초 지점(`x=30`)부터 60초 지점(`y=60`)까지 재생합니다.

```text
youtube: https://www.youtube.com/watch?v=MMmOLN5zBLY&start=30&end=60
```

보다 직관적인 "시간:분:초" 표기로도 지정할 수 있습니다.

```text
youtube: https://www.youtube.com/watch?v=MMmOLN5zBLY&start=0:30&end=1:00
```

**참고:** YouTube 동영상을 슬라이드에 표시하려면 위 명령어가 해당 슬라이드에서 유일한 블록이어야 합니다. 여러 블록을 기술하면 동영상 자체는 표시되지 않고 동영상 링크가 사용됩니다.

#### MP4 동영상 삽입

웹에 업로드된 MP4 형식의 동영상 파일 URL을 지정하여 슬라이드에 표시할 수 있습니다. 동영상은 슬라이드 전체에 가능한 한 크게 확대됩니다.

```text
video: url_to_your_video_file.mp4
```

동영상의 URL에 시작 지점과 종료 지점을 지정할 수 있습니다. 동영상 시작 시점부터의 경과 초수로 표현하며 `#t=x,y` 형식으로 URL에 옵션을 추가합니다. 다음 예시는 동영상의 처음(`x=0`)부터 5초 후(`y=5`)까지 재생합니다.

```text
video: url_to_your_video_file.mp4#t=0,5
```

**참고:** MP4 동영상을 슬라이드에 표시하려면 위 명령어가 해당 슬라이드에서 유일한 블록이어야 합니다. 여러 블록을 기술하면 동영상 자체는 표시되지 않고 동영상 링크가 사용됩니다.

#### MP3 오디오 삽입

웹에 업로드된 MP3 형식의 오디오 파일 URL을 지정하여 슬라이드에 표시할 수 있습니다.

```text
audio: url_to_your_audio_file.mp3
```

MP3 오디오의 URL에 시작 지점과 종료 지점을 지정할 수 있습니다. 오디오 시작 시점부터의 경과 초수로 표현하며 `#t=x,y` 형식으로 URL에 옵션을 추가합니다. 다음 예시는 오디오의 처음(`x=0`)부터 5초 후(`y=5`)까지 재생합니다.

```text
audio: url_to_your_audio_file.mp3#t=0,5
```

**참고:** 하나의 슬라이드에 여러 오디오 파일을 포함할 수 있습니다. 슬라이드에 모든 블록이 들어가지 않으면 자동으로 스크롤됩니다.

### 6. 추가 기능

#### 텍스트 음성 변환(TTS)

일반 단락의 문장은 브라우저에서 사용 가능한 TTS 엔진으로 음성 재생할 수 있습니다. 사용 가능한 언어와 음성 종류는 OS와 브라우저에 따라 다릅니다.

하이라이트된 콘텐츠에 TTS를 사용할 수 있을 때 프레젠테이션 화면 왼쪽 상단에 스피커 아이콘이 나타납니다. 이 아이콘을 클릭하거나 키보드의 `.` 키를 누르면 TTS 음성이 재생됩니다. 다시 클릭하거나 키를 누르면 재생이 중지됩니다.

TTS 재생 중에는 현재 읽히고 있는 단어가 노란색 배경으로 하이라이트됩니다. 이 단어 수준의 하이라이트를 통해 청중은 읽기 위치를 쉽게 따라갈 수 있습니다. 재생이 끝나거나 중지되면 하이라이트가 자동으로 제거됩니다.

**참고:** 단어 수준 하이라이트는 브라우저의 `onboundary` 이벤트 지원에 의존합니다. Chrome과 Edge에서는 잘 작동하지만 모든 브라우저에서 사용 가능한 것은 아닙니다.

#### 메모지(스티키 노트)

프레젠테이션 중 떠오르는 내용을 메모지에 작성하여 청중에게 보여줄 수 있습니다.

프레젠테이션 화면 왼쪽 상단의 메모지 아이콘을 클릭하여 메모지를 표시하거나 숨길 수 있습니다. 메모지의 내용은 슬라이드가 바뀌어도 유지됩니다.

#### 레이저 포인터

마우스 커서의 형태/색상을 일반 커서에서 레이저 포인터 스타일의 색상 점으로 변경할 수 있습니다. 화면 왼쪽 상단의 ● 아이콘을 클릭하여 모드를 전환합니다.

변경된 마우스 포인터의 색상은 설정 패널에서 선택한 "강조 색상"에 의해 결정됩니다.


#### 자동 프레젠테이션

화면 왼쪽 상단의 마법 지팡이 아이콘을 클릭하세요. 현재 슬라이드의 다음 프래그먼트부터 전체 프레젠테이션의 마지막 프래그먼트까지 텍스트 읽기, 동영상 재생 등이 자동으로 차례대로 실행됩니다.

#### 자동 저장

입력 텍스트와 폼 설정은 입력하는 동안 브라우저의 로컬 스토리지에 자동으로 저장됩니다. 페이지를 다시 방문하면 이전 작업 내용이 자동으로 복원됩니다. **텍스트 지우기** 버튼을 클릭하면 에디터 내용과 저장된 데이터가 모두 초기화됩니다.

#### HTML 다운로드

**HTML 다운로드** 버튼을 클릭하면 프레젠테이션을 독립 실행형 HTML 파일로 내보낼 수 있습니다. 파일에는 슬라이드 내용, 설정, 스타일이 모두 포함되어 있어 Paradocs 웹사이트에 접속하지 않아도 모든 브라우저에서 열 수 있습니다. 외부 라이브러리(jQuery, Reveal.js 등)는 CDN을 통해 로드됩니다.

**참고:** 내보낸 파일에서 YouTube 동영상 삽입은 클릭 가능한 섬네일 링크로 자동 변환됩니다. 이는 로컬 HTML 파일에서는 브라우저 보안 제한으로 인해 YouTube iframe 삽입이 작동하지 않기 때문입니다. 섬네일을 클릭하면 YouTube에서 동영상이 열립니다.

#### URL 공유

슬라이드와 프래그먼트를 탐색하면 프레젠테이션 URL이 자동으로 업데이트됩니다. 브라우저 주소창에서 URL을 복사하여 특정 위치를 공유할 수 있습니다(예: `#/0/1/3`). 공유된 URL을 열면 해당 슬라이드와 프래그먼트로 바로 이동합니다.

#### 다크 모드(색상 반전)

**색상 반전** 체크박스를 선택하면 다크 모드로 전환됩니다. 메모지, 주석 팝업, 마커, 슬라이드 번호, 내비게이션 컨트롤 등 모든 프레젠테이션 요소가 어두운 배경에 맞게 색상이 조정됩니다.
